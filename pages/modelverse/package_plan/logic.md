# 套餐计费逻辑

本页说明 优云智算 套餐（按量包/包月）在计费与扣除上的规则，套餐说明见[套餐包快速上手](/modelverse/package_plan/package.md)，套餐用量统计见[套餐统计](/modelverse/package_plan/stats.md)。

## 1. 计量方式

Coding Plan 以**调用次数**为计量单位，与请求的 Token 数量无关。

不同模型按**额度倍率**消耗套餐次数

> 示例：kimi-k2.6 倍率为5×，则调用 1 次 kimi-k2.6，套餐额度扣减 5 次。

---

## 2. 额度重置机制：滚动 5 小时窗口

Coding Plan 采用**固定 5 小时窗口**刷新额度，窗口以准点为起点计算。

### 窗口示例

| 窗口起始 | 窗口结束 |
|---------|---------|
| 15:00 | 20:00 |
| 20:00 | 01:00（次日） |
| 01:00 | 06:00 |

- 当前窗口内额度耗尽时，API 返回 `429` 错误，响应中包含下次重置时间
- 等待窗口重置后即可继续使用，无需任何操作

---

## 3. 三重额度约束

调用额度受以下三层限制，**任意一层耗尽均会触发限流**：

|约束维度 | Mini 迷你版 | Lite 入门版 | Basic 基础版 | Pro 增强版 | Max 高级版 | Ultra 畅享版 |
|---|---|---|---|---|---|---|
| 5小时窗口 | 约 200 次 | 约 400 次 | 约 800 次 | 约 2,000 次 | 约 3,000 次 | 约 4,000 次 |
| 每周总量 | 约 650 次 | 约 1,300 次 | 约 2,600 次 | 约 6,500 次 | 约 10,500 次 | 约 13,500 次 |
| 每月总量 | 约 1,900 次 | 约 3,800 次 | 约 7,600 次 | 约 19,000 次 | 约 31,000 次 | 约 39,000 次 |

---

## 4. 续费规则

在有效期内重复购买**同一档位**套餐，**有效期顺延 1 个月，额度上限不变**。

**示例：**
> Lite 有效期至 4月26日，再次购买 Lite → 有效期延至 5月26日。

> 若需要更多额度，应选择**升级到 Pro**，而非多次购买 Lite。

---

## 5. 升级规则：Lite → Pro（差价折算）

从 Lite 升级到 Pro 时，按剩余天数折算 Lite 已用价值，**只需补缴差额**：

```
补缴金额 = (¥499 - ¥99) * (剩余天数 / 30)
结果向上取整至分
```

**示例（剩余 15 天）：**

| 项目 | 金额 |
|------|------|
| 补缴金额计算 | 补缴金额 = (¥499 - ¥99) * (剩余天数 / 30) |
| **本次需补缴** | **¥200** |

升级后立即生效：有效期沿用旧套餐到期时间。

---

## 6. 额度耗尽处理

Coding Plan **不支持超额自动计费**，额度耗尽后专属 Key 直接停止服务：

- API 返回 `429` 错误，响应中包含窗口重置时间
- 月度总量耗尽，等下月或升级套餐
- 如需立即继续调用，可**手动切换到普通按量 API Key**，按量计费不中断

---

## 7. 补充说明

| 场景 | 处理方式 |
|------|---------|
| 并发超限 | 返回报错，减少并发后重试 |
| 调用套餐范围外的模型 | 套餐专属 Key 仅支持套餐内模型；如需调用其他模型，请切换至普通按量 API Key |
| 扣费异常排查 | 在「当前套餐 → 调用明细」查看每条记录的套餐 ID 与消耗次数 |
